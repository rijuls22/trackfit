# TrackFit — Vercel Deployment Guide

## What you're deploying
- Frontend: public/index.html
- Backend: 3 API serverless functions (api/workouts.js, api/goals.js, api/feedbacks.js)
- Database: Vercel Postgres (free, powered by Neon)

---

## STEP 1 — Create a GitHub account (if you don't have one)
1. Go to https://github.com
2. Click "Sign up" — use your school email
3. Verify your email

---

## STEP 2 — Upload the project to GitHub
1. Go to https://github.com/new
2. Name your repo: trackfit
3. Set it to Public
4. Click "Create repository"
5. On the next screen, click "uploading an existing file"
6. Drag and drop the ENTIRE trackfit folder contents:
   - vercel.json
   - package.json
   - public/index.html
   - api/workouts.js
   - api/goals.js
   - api/feedbacks.js
7. Click "Commit changes"

---

## STEP 3 — Create a Vercel account
1. Go to https://vercel.com
2. Click "Sign Up"
3. Choose "Continue with GitHub" — this links them automatically

---

## STEP 4 — Import your project to Vercel
1. On your Vercel dashboard, click "Add New Project"
2. Find your "trackfit" GitHub repo and click "Import"
3. Leave all settings as default
4. Click "Deploy"
5. Wait ~60 seconds — Vercel will build and deploy your site
6. You'll get a live URL like: https://trackfit-abc123.vercel.app

---

## STEP 5 — Create the Postgres database
1. On your Vercel project dashboard, click the "Storage" tab
2. Click "Create Database"
3. Choose "Postgres" (powered by Neon) — it's FREE
4. Name it: trackfit-db
5. Choose region: closest to you (e.g. Sydney = ap-southeast-2)
6. Click "Create"

---

## STEP 6 — Connect the database to your project
1. After creating the database, click "Connect Project"
2. Select your trackfit project
3. Click "Connect"
4. Vercel automatically adds the DATABASE_URL environment variable to your project

---

## STEP 7 — Redeploy to apply the database connection
1. Go to your project → "Deployments" tab
2. Click the three dots (...) on the latest deployment
3. Click "Redeploy"
4. Wait ~30 seconds

---

## STEP 8 — Test it!
1. Visit your live URL (e.g. https://trackfit-abc123.vercel.app)
2. Log a workout
3. Refresh the page — the workout should still be there (it's in the database!)
4. Open the site on your phone — it works there too, and shows the same data

---

## Your File Structure (must match exactly)
trackfit/
├── vercel.json          ← tells Vercel how to route requests
├── package.json         ← lists the @vercel/postgres dependency
├── public/
│   └── index.html       ← the full TrackFit frontend
└── api/
    ├── workouts.js      ← GET/POST/DELETE workouts
    ├── goals.js         ← GET/POST/PUT/DELETE goals
    └── feedbacks.js     ← GET/POST feedbacks

---

## Troubleshooting

PROBLEM: "Error saving" when logging a workout
FIX: Make sure you completed Steps 5–7 (database not connected yet)

PROBLEM: Site loads but data doesn't save
FIX: Go to Vercel → your project → Settings → Environment Variables
     Make sure POSTGRES_URL exists. If not, redo Step 6.

PROBLEM: "Function timeout" error
FIX: This is rare on Vercel free tier. Just try again — it's a cold start.

PROBLEM: Changes not showing after editing files
FIX: Any time you push new files to GitHub, Vercel auto-redeploys within ~60 seconds.

## How the data flows (for your documentation)

User browser → fetch('/api/workouts') → Vercel Serverless Function → Vercel Postgres DB
                                      ↑
                               (runs in Node.js on Vercel's servers)

No localStorage is used anywhere. All data lives in the Postgres database on Vercel's servers.
